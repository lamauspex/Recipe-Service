# Postman коллекции для тестирования

## Импорт коллекции

1. Открой **Postman**
2. Нажми **Import** (кнопка в левом верхнем углу)
3. Перетащи файл `User Service.postman_collection.json` в окно импорта
4. Нажми **Import**

## Настройка переменных

После импорта нужно настроить переменную `base_url`:

1. Нажми на **шестерёнку** (правый верхний угол) → **Environments**
2. Выбери или создай окружение (например, `Development`)
3. В разделе **Variable** проверь:
   - `base_url` = `http://localhost:8001` (или `http://localhost:8000` если запускаешь локально без Docker)
4. Нажми **Save**

## Использование

### 1. Регистрация нового пользователя
1. Выбери **Register** в коллекции
2. При желании измени email/password в Body
3. Нажми **Send**
4. Скопируй email для следующего шага

### 2. Логин
1. Выбери **Логин**
2. Вставь зарегистрированный email и пароль
3. Нажми **Send**
4. **Скопируй `refresh_token`** из ответа
5. Вставь его в переменную `refresh_token` (шестерёнка → Environments)

### 3. Обновить токен
1. Выбери **Обновить токен**
2. Убедись что `{{refresh_token}}` подставлен в Body
3. Нажми **Send**
4. Получишь новый `access_token`

### 4. Выход
1. Выбери **Выход**
2. Нажми **Send**
3. Refresh токен будет инвалидирован

## Переменные

| Переменная | Описание | Пример |
|------------|----------|--------|
| `base_url` | Базовый URL сервиса | `http://localhost:8001` |
| `access_token` | JWT токен доступа | `eyJ...` |
| `refresh_token` | Токен обновления | `eyJ...` |

## Тестирование без Docker

Если запускаешь сервис локально (не через Docker):

1. Измени `base_url` на `http://localhost:8000`
2. Запусти сервис: `python -m backend.service_user.main`

## Дополнительно

- **History** — история всех запросов (левая панель)
- **Collections** — сохранённые наборы запросов
- **Environments** — наборы переменных для разных окружений
